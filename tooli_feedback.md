[x] Remove all emoji from the help files — [#195](https://github.com/weisberg/tooli/issues/195) (closed as [#192](https://github.com/weisberg/tooli/issues/192))
[x] **Security sanitizer strips generic type syntax from heading text** — When the `headers` command was used to search `TOOLI_DEV_GUIDE.md`, headings containing bracket-parameterized types (e.g. `StdinOr[T]`, `SecretInput[T]`, `TooliResult[T]`) were returned as `[redacted]` instead of their actual text. The output sanitizer in `tooli/security/sanitizer.py` appears to be matching `[T]` as a markdown link pattern and redacting it. This corrupts heading extraction output for any documentation that uses generic type notation in headings. Affected lines in the dev guide: 327, 343, 477. Expected behavior: sanitizer should only redact shell injection patterns, not square-bracket text that lacks a preceding `(url)` component. — [#196](https://github.com/weisberg/tooli/issues/196) (closed as [#193](https://github.com/weisberg/tooli/issues/193))
[x] **Global `--output` / `-o` flag conflicts with user-defined `--output` command parameters** — The globally injected `-o`/`--output` flag (which controls output mode: auto, json, jsonl, text, plain) silently takes precedence over any command parameter also named `--output`, making it impossible to define an `--output <file>` option in a command. The conflict produces a confusing error: `'myfile.png' is not one of 'auto', 'json', 'jsonl', 'text', 'plain'`. Workaround applied in this repo: renamed all output-path parameters to `--out-file`. Suggested fix: either document the reserved parameter names so developers know to avoid them, or detect the conflict at command registration time and raise a clear error (e.g. `"Parameter name 'output' conflicts with the global --output flag; use a different name"`). Affected global flags to document as reserved: `--output`/`-o`, `--json`, `--jsonl`, `--text`, `--plain`, `--dry-run`, `--yes`/`-y`, `--force`, `--quiet`/`-q`, `--verbose`/`-v`, `--timeout`, `--idempotency-key`. — [#197](https://github.com/weisberg/tooli/issues/197) (closed as [#194](https://github.com/weisberg/tooli/issues/194))
[x] **[CRITICAL — fixed in v6.6.0] CLI crashes on all `T | None` optional parameters — `ValueError: T | None is not callable`** — v6.5.0's `_build_parser()` passed raw `T | None` union types directly to argparse as `type=`. Fixed in v6.6.0. — [#203](https://github.com/weisberg/tooli/issues/203)
[x] **[URGENT — fixed in v6.6.0] v6.5.0 breaking change: `triggers`, `anti_triggers`, `rules` removed from `Tooli()` constructor** — Restored in v6.6.0; all three fields accepted again. — [#202](https://github.com/weisberg/tooli/issues/202)

[x] **No environment variable / `--yes` flag to suppress `Destructive` confirmation prompts** — Resolved in v6.6.0: `Destructive` commands no longer prompt for confirmation in any context; `--yes` flag removed entirely. Commands run without confirmation overhead. — [#199](https://github.com/weisberg/tooli/issues/199)
